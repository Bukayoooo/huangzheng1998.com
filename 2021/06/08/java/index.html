Java的内存分配

Java中两个重要的内存：占内存与堆内存

栈内存：用来存放函数(方法)和局部变量，局部变量存储在栈内存中，数据在函数执行完毕后自动释放。

堆内存：用来存放对象和数组，由程序员来分配和释放，程序结束后有系统回收；C++中通常使用free方法来释放内存，Java中有垃圾回收机制，无需程序员手动释放内存；使用new方法创建的数据，都存放在堆区。

C++执行时，内存会分为四大区：代码区、全局区、栈区、堆区；

代码区用来存放二进制代码，全局区用来存放全局变量、静态变量、常量

Java执行时，内存分为五个区：寄存器区、本地方法栈区、方法区、栈区、堆区

Java函数重载

概念：同一个函数名，参数不同、类型不同、数量不同表示的函数也不相同

  1. 函数需要在一个类中
  2. 函数需要有相同的函数名
  3. 函数的形参不同、形参的类型不同、或形参的数量不相同

注意：函数重载仅对函数名与函数形参敏感，与返回值无关，返回值类型不同的函数不是函数重载

Java的类与对象

类的定义

在C++中，主函数与类是分开的。一个程序里面可以写很多个类，但是在Java里，一个程序就是一个类，类里面写主函数，因此Java程序中一个程序只能写一个类，且类名就是程序名

定义一个学生类

对象的使用

将类实例化就是具体的对象

创建对象的格式：类名 对象名 = new 类名();

这里的new 类名();是类里的默认构造函数，即使类中没有定义，程序会自动写一个默认构造函数。

PS：这里与C++不相同，C++可以直接实例化一个对象，不用new申请内存也可以，但是Java不行，直接实例化会报错。

成员变量与局部变量

成员变量：类中方法外定义的变量

局部变量：在类中方法内部或方法声明中定义的变量

区别：1. 成员变量存储于堆内存中，局部变量存储于栈内存中

        2. 成员变量随着对象的存在而存在，随着对象的消失而消失
           3. 成员变量有默认初始化值，局部变量没有默认初始化值，必须先定义，赋值才能使用

private与this关键字

我勒个去，Java中竟然不能写连小于0<age<120!!!

C++代码

private: 私有成员变量，只有类内可以修改，类外不可修改；类外若修改，可通过设置类内的方法来实现；

this: this修饰的变量代指成员变量，主要作用是区分局部变量与成员变量的崇明问题

  • 函数的形参如果和成员变量重名，不带this修饰的变量指的是形参，而不是成员变量
  • 函数的形参没有和成员变量重名，不带this修饰的变量指的是成员变量，而不是形参

this指针代表当前调用方法的引用，哪个对象调用的方法，this就代表哪一个对象；Java的this指针与C++的this指针类似，只不过C++中使用this->name, Java中使用this.name.也就是Python中的self指针。

构造函数

构造函数是一种特殊的函数，在实例化对象时，自动调用这个函数。也就是Python中的 def init 函数，顾名思义，其完成的功能就是成员变量的初始化。
